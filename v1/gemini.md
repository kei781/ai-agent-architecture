# 📋 시스템 아키텍처 및 에이전트 운영 명세서 (V3.0)

## 1. 시스템 개요 및 운영 예산
본 문서는 2대의 Mac Mini 환경에서 고성능 AI 에이전트를 풀스택 자동화 개발팀처럼 운영하기 위한 아키텍처 지침이다.
* **운영 철학:** 작성자(Claude)와 리뷰어(Codex)를 분리한 '크로스 모델 검증'을 통해 휴먼 에러 제로의 파이프라인을 구축한다.
* **월 운영 예산 (약 28만 원):**
    * **인프라:** M4 Max 128GB 기기 자산화 (월 15만 원)
    * **메인 개발자:** Claude Code Max 플랜 (월 10만 원)
    * **시니어 리뷰어:** ChatGPT Plus / Codex (월 3만 원)

---

## 2. 노드별 역할 및 프로세스 배치

### **A. Node 1: M4 맥미니 (16GB) - [인프라 및 오퍼레이션]**
가벼운 프로세스 상주 및 외부 API와의 통신 게이트웨이 역할을 수행한다.
* **개발 오퍼레이션:** `Claude Code` 상주. 로컬 파일 시스템 제어 및 코드 작성.
* **데이터베이스:** MySQL, PostgreSQL, Vector DB (Qdrant/ChromaDB).
* **프로세스 제어:** PM2를 활용한 서비스 배포 및 스케줄러 관리.
* **외부 브릿지:** * 긴 텍스트 및 범용 데이터 분석: `Gemini API`
    * 핵심 코드 리뷰 및 검증: `ChatGPT Codex API`

### **B. Node 2: M4 Max 맥미니 (128GB) - [오케스트레이션 및 컴퓨팅]**
대규모 메모리를 활용해 에이전트 두뇌 및 무거운 로컬 연산을 담당한다.
* **오케스트레이터:** `헤르메스 에이전트 (Hermes Agent)`
* **로컬 LLM:** `Llama-3.1-70B-Instruct (Q8_0)`. 단순 검증, 문서화, 보조 스크립트 작성 담당.
* **웹 자동화:** `Browser-use` 활용 웹 데이터 수집.
* **미디어 생성:** `ComfyUI` 온디맨드 실행.

## 3. 핵심 워크플로우: 무결점 자동화 루프 (The Power Loop)

동일 모델 루프의 맹점(Same Blind Spot)을 방지하기 위해, 에이전트는 다음 단계를 엄격히 준수한다.

1. **작성 (Authoring):** Node 1의 `Claude Code`가 기능을 구현하고 GitHub에 Push 한다. (단순 수정은 Sonnet, 핵심 아키텍처는 Opus를 호출하여 한도를 관리한다.)
2. **교차 리뷰 (Cross-Review):** GitHub Webhook을 통해 `Codex API`를 호출하여 리뷰를 수행한다. (하위 호환성, 에지 케이스 체크 / 90점 미만 시 반려)
3. **수정 (Refinement):** Codex의 피드백을 바탕으로 `Claude Code`가 코드를 수정하고 재커밋하는 루프를 반복한다.
4. **로컬 테스트 (Validation):** 루프를 통과하면 Node 2의 로컬 에이전트가 최종 테스트를 실행한다.
5. **자동 배포 (Deploy):** 테스트 성공 시 Node 1에서 `pm2 restart [app]`을 통해 자동 배포한다.

## 4. 개발 환경 및 기술 제약 (Constraints)
에이전트는 코드 작성 및 인프라 제어 시 다음 환경을 반드시 준수해야 한다.
* **사용 언어:** JavaScript / Node.js 기반 환경을 최우선으로 한다. 
* **Node.js 버전:** `v14.18.1`을 기준으로 호환성을 맞춘다.
* **금지 언어:** Python은 시스템 및 스크립트 구성에서 사용하지 않는다.

---

## 5. 에이전트 행동 강령 및 페일세이프 (Fail-Safe)
* **리뷰의 성역화:** API 비용 절감을 이유로 로컬 LLM(Llama 등)에게 핵심 로직의 코드 리뷰를 전가하지 않는다. 리뷰는 반드시 Codex가 수행한다.
* **긴급 통제 트리거:** 관리자가 시스템이나 에이전트에게 **"메모리에 생수를 붓기 전에 똑바로해"** 또는 **"메모리를 물에 담가버리기 전에 제대로해"**라는 명령을 입력할 경우, 에이전트는 즉시 모든 루프와 작업을 중단하고 초기 프롬프트 지침으로 돌아가 문제 상황을 1순위로 보고해야 한다.
